# **Landsat Spectral Complexity Data Dictionary & Navigation Guide**

## **1\. Overview**

This document describes the structure, contents, and metadata of Landsat Level-2 Science Product (L2SP) image stacks that have been processed by the spectral\_complexity\_calculations.py pipeline.

The file strictly adheres to the HDF-EOS5 grid specification and aligns with the Climate and Forecast (CF) Metadata Conventions. This enables seamless time-series analysis of localized spectral hypervolumes, endmember extraction, and background-masked anomaly Z-scores while maintaining rigorous spatial metrology.

**Note on Failure Handling & Data Integrity:** Following strict data science directives, synthetic fill values (e.g., 0.0) are *not* used to represent missing or masked data in floating-point analytical arrays. Radiometrically invalid pixels, cloud-covered areas, and frames with completely invalid background models are rigorously populated with NaN (Not a Number) to prevent artificial skewing of downstream Neural Network feature spaces.

## **2\. HDF5 Hierarchy & Base Path**

All temporal data, spectral data, and calculated indices are stored under a single, unified HDF-EOS5 group path.

* **Base Target Path:** /HDFEOS/GRIDS/LANDSAT/Data Fields

### **Dimensional Conventions & Memory Layout**

* **Temporal Dimension (T):** All datasets share a primary temporal dimension (num\_frames). Time alignment is perfectly 1:1 across all datasets.  
* **Spatial Dimensions (H, W):** Spatial datasets share Height (rows) and Width (columns).  
* **Spectral/Endmember Dimensions (B, N):** num\_bands and num\_endmembers.  
* **Memory Layout Constraint:** Unless otherwise specified, multi-dimensional spatial arrays are stored in a **Band Sequential (BSQ)** memory layout (e.g., \[T, B, H, W\]).

## **3\. Dataset Specifications**

### **3.1 Pre-Existing Source Datasets (Inputs)**

These datasets are generated by the upstream Landsat Stacker and serve as the baseline for the complexity calculations.

* **surface\_reflectance**  
  * **Shape:** \[T, B, H, W\] (float32)  
  * **Description:** Bottom-of-Atmosphere (BOA) reflectance scaled 0.0 to 1.0.  
  * **Attributes (Crucial for Spatio-Temporal Metrology):**  
    * acquisition\_time: 1D Array of UNIX timestamps (UTC) corresponding to the T axis.  
    * spacecraft\_id: 1D Array denoting the platform (e.g., 'LANDSAT\_8' vs 'LANDSAT\_9').  
    * wavelengths: 1D Array mapping the B axis to physical center wavelengths (in μm).  
    * GeoTransform: 6-element array defining the GDAL affine transformation matrix.  
    * spatial\_ref: String containing the Well-Known Text (WKT) Coordinate Reference System (e.g., EPSG WGS84 UTM).  
* **ortho\_visual**  
  * **Shape:** \[T, C, H, W\] (uint8)  
  * **Description:** High-resolution true-color rendering (where C is RGB or RGBA) of the scene used for context, visualization, and cross-correlation tracking.  
  * **Format Constraints:** Stored strictly in BSQ format.  
  * **Masking Note:** In RGBA cubes (C=4), the alpha channel strictly defines the valid swath footprint. Pixels where Alpha \> 0 are valid data; Alpha \= 0 represents the NoData background.  
* **QUALITY\_L1\_PIXEL**: \[T, H, W\] (uint16). USGS Landsat Collection 2 QA Bitmask.  
* **QUALITY\_L2\_AEROSOL**: \[T, H, W\] (uint8). LaSRC Aerosol classification.  
* **RADIOMETRIC\_SATURATION**: \[T, H, W\] (uint16/uint8). Radiometric saturation flags.

### **3.2 Global Endmember Datasets**

Derived using the Maximum-Distance (MaxD) simplex volume algorithm across the entire valid frame.

* **frame\_endmembers**  
  * **Shape:** \[T, B, N\] (float32)  
  * **Description:** The spectral signatures of the N most spectrally extreme pixels in the frame.  
  * **Attributes:** Normalization (e.g., 'bandCount'), num\_endmembers (7).  
* **frame\_endmember\_indices**  
  * **Shape:** \[T, N\] (int32)  
  * **Description:** The 1D flattened spatial indices of the extracted endmembers.  
* **frame\_endmember\_volumes**  
  * **Shape:** \[T, N\] (float32)  
  * **Description:** The expanding volume curve of the spectral convex hull as each of the N endmembers is sequentially added to the simplex.

### **3.3 Spatial Complexity & Windowed Datasets**

Mapped spatial calculations representing localized spectral heterogeneity.

* **sliding\_volume\_map**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Convex hull volume of spectral data within a sliding 3x3 window. Pixels inside windows containing *any* masked clouds/shadows evaluate to NaN.  
  * **Attributes:** tile\_size (3), sliding\_stride (1), gram\_type.  
* **sliding\_volume\_map\_5x5**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Convex hull volume using a larger 5x5 spatial context window.  
  * **Attributes:** tile\_size (5).  
* **sliding\_volume\_map\_7x7**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Convex hull volume using a 7x7 spatial context window.  
  * **Attributes:** tile\_size (7).  
* **msd\_map**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Local Mean Spectral Distance (MSD) across a 3x3 sliding window.

### **3.4 Vegetation & Urban Indices**

Standardized spectral indices computed on valid surface reflectance.

* **ndvi\_map**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Normalized Difference Vegetation Index. Bound between \[-1.0, 1.0\]. Invalid pixels evaluate to NaN.  
  * **Attributes:** red\_idx (3), nir\_idx (4).  
* **ndbi\_map**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Normalized Difference Built-up Index. Bound between \[-1.0, 1.0\].  
  * **Attributes:** swir\_idx (5), nir\_idx (4).

### **3.5 Anomaly Detection Datasets**

* **sliding\_volume\_z\_score\_masked**  
  * **Shape:** \[T, H, W\] (float32)  
  * **Description:** Global Spectral Complexity Z-score mapping. The statistical background model (mean, std dev) is rigorously calculated *only* from clear, radiometrically valid pixels. Z-scores are then applied to all geometrically valid volumes. Frames with zero valid background pixels return entirely NaN.  
  * **Crucial Masking Attributes (Embedded in Dataset):**  
    * MASKING\_APPLIED: True  
    * SUN\_ELEVATION\_THRESHOLD: 30 (Excludes frames with high Solar Zenith Angle shadows).  
    * CLOUD\_DILATION: 2 (Buffer pixels around detected clouds).  
    * QA\_REJECT\_MASK: 63 (0b111111 \- Rejects Dilated Cloud, Cirrus, Cloud, Shadow, Snow).  
    * AEROSOL\_ACCEPT\_LEVEL: 'medium' (Enforces theoretical LaSRC physical limitations by rejecting AOT \>= 0.3).

## **4\. AI Agent / Developer Navigation Guide (Python)**

To effectively read and iterate over this stack, use the following h5py access pattern. Do not hardcode the number of frames or bands; dynamically fetch them from the .shape attribute.

**Critical Metrology Warning:** When extracting visual or reflectance arrays for use in standard computer vision libraries (e.g., skimage, OpenCV), you **must** transpose the array from Band Sequential (BSQ) \[C, H, W\] to Band Interleaved by Pixel (BIP) \[H, W, C\] to prevent silent mathematical broadcasting errors.

import h5py  
import numpy as np

\# Recommended Access Pattern  
file\_path \= "LANDSAT\_Stack\_Complexity\_Analysis.h5"  
with h5py.File(file\_path, 'r') as h5:  
    \# 1\. Navigate to the core Data Fields group  
    data\_grp \= h5\['HDFEOS/GRIDS/LANDSAT/Data Fields'\]  
      
    \# 2\. Extract datasets by reference (prevents loading entire gigabyte arrays into RAM)  
    z\_scores \= data\_grp\['sliding\_volume\_z\_score\_masked'\]  
    ndvi \= data\_grp\['ndvi\_map'\]  
      
    \# 3\. Read masking metadata directly from the dataset attributes  
    \# This guarantees your downstream script knows exactly what filtering rules were applied  
    cloud\_buffer \= z\_scores.attrs.get('CLOUD\_DILATION', 0\)  
    print(f"Dataset generated with cloud dilation of {cloud\_buffer} pixels.")  
      
    num\_frames, height, width \= z\_scores.shape  
      
    \# 4\. Chronological Iterator (Memory Efficient)  
    for t in range(num\_frames):  
        \# Load single time-slice into memory  
        frame\_z \= z\_scores\[t, :, :\]  
        frame\_ndvi \= ndvi\[t, :, :\]  
          
        \# Example analytical filter: Find highly anomalous, non-vegetated pixels  
        valid\_data\_mask \= \~np.isnan(frame\_z) & \~np.isnan(frame\_ndvi)  
        anomalies \= valid\_data\_mask & (frame\_z \> 3.0) & (frame\_ndvi \< 0.2)  
          
        if np.any(anomalies):  
            print(f"Found {np.sum(anomalies)} anomalies in frame {t}")
