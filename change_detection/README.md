# Anomaly Detection in Temporal Series Using the Spectral Complexity Metric
The overall goal of this project is to characterize the utility of the spectral complexity metric I have developed in my qualifying exam work as a feature for anomaly detection in a temporal series of multispectral or hyperspectral images. The method I have in mind was to use PyTorch to train a transformer on my dataset and identify when the feature identifies a change from a baseline. This blog post (https://www.geeksforgeeks.org/deep-learning/transformer-for-time-series-forecasting/) outlines a similar concept that I have been trying to emulate. 

In class it was recommended to use a simple statistical tool as a baseline to compare neural network solutions to in case the more complicated solution is worse in performance. I have also implemented a Continuous Change Detection (CCD) algorithm that models the time series of each pixel as a Fourier harmonic series. The CCD algorithm is often used with Landsat imagery like I am processing for my complexity metric and is well suited for the size of the dataset I am working with. 

The input to the algorithm is an HDF5 file that contains the relevant datasets for processing all aligned to a common aligned spatial grid. The datasets used are ortho_visual, sliding_volume_map, and sliding_volume_z_score. The ortho_visual dataset is a 3 band color representation of the multispectral dataset for viewing identified pixels in a scene. The dataset sliding_volume_map is the local spectral complexity metric that I developed as the focus of my QE work. The sliding_volume_map is a measure of the change of spectral values in a 3x3 local neighborhood. sliding_volume_map values follow a log-normal distribution with a mean of 5e-5 and variance of 2e-8. This log-normal distribution is not well suited as an input for ML algorithms. The dataset sliding_volume_z_score is the z-score values calculated from sliding_volume_map to better align with this computer vision project. The time series is formed by the Landsat images captured over Rochester from 2015 through 2025 with frame exclusions for cloud or snow cover. 

The CCD algorithm calculates the best fit solution for the 6 term harmonic series of [Bias, Slope, Cos(1x), Sin(1x), Cos(2x), Sin(2x)] for the sliding_volume_z_score time series for each pixel. The training is performed on data from 2015 through 2023. The evaluation period for anomalies is performed on 2024-2025. If a selected number (currently 3) of sequential values in the time series are outside a number of standard deviations (currently 3) from the harmonic series prediction it is flagged as an anomaly. 

The output of the training and anomaly detection is saved for a separate viewer script to retrieve and display. The pixel locations are displayed on the color image of the scene with an interactive pixel selection to display the time series values for that pixel along with the harmonic series prediction range.
